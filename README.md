# 🌡️ MQTT 环境监控系统

一个基于 MQTT 的实时环境数据监控 Web 应用，支持温度、湿度、风速、光照强度等多种环境参数的实时采集、展示和历史数据分析。

## 📋 项目概述

该系统通过 MQTT 协议与物联网设备通信，实时采集环境数据，并在浏览器中以可视化图表形式展示。支持灵活的 MQTT 配置、数据实时更新和历史趋势分析。

### 核心特性

- ✅ **实时 MQTT 连接** - 支持 WebSocket Secure (wss://) 协议
- ✅ **多参数监控** - 温度、湿度、风速、光照强度
- ✅ **动态数据图表** - 使用 ECharts 5.4.3 进行实时绘制
- ✅ **灵活配置界面** - 动态修改 MQTT 连接参数
- ✅ **响应式设计** - 完美支持桌面、平板、手机设备
- ✅ **现代化 UI** - 渐变设计、流畅动画、专业视觉
- ✅ **加载反馈** - 页面加载时展示动画指示器
- ✅ **安全防护** - 严格的内容安全策略（CSP）

## 📊 功能详解

### 1. 实时数据展示

**数据卡片** - 四个大卡片展示实时数据：
- 🌡️ **温度** - 显示当前环境温度（℃）
- 💧 **湿度** - 显示当前环境湿度（%）
- 💨 **风速** - 显示当前风速（m/s）
- 💡 **光照强度** - 显示当前光照强度（lux）

**数据更新动画** - 当数据变化时：
1. 数值放大至 1.12 倍
2. 文字闪现蓝色
3. 0.8s 内恢复正常

### 2. 动态数据图表

**主图表** - 绘制三条时间序列曲线：
- 温度趋势（实线）
- 湿度趋势（实线）
- 风速趋势（实线）
- 包含图例、网格、滚动条等交互功能

**光照图表** - 专独图表展示：
- 光照强度变化趋势
- 面积图填充效果
- 独立的数据缩放

**图表特性**：
- 响应式高度（根据设备自适应）
- 时间轴标签旋转 30°-45°（提升可读性）
- 数据点最多保留 20 个（滑动窗口）
- 设备自适应字体大小

### 3. MQTT 连接配置

**配置弹窗** - 点击菜单进入配置：

| 配置项 | 说明 | 默认值 | 备注 |
|--------|------|--------|------|
| **服务器地址** | WebSocket Secure 地址 | wss://... | 必填，包含端口 |
| **客户端ID** | 唯一标识符 | 自动生成 | 可自定义 |
| **订阅主题** | MQTT 主题 | environment/data | 必填，支持正则验证 |
| **用户名** | MQTT 认证用户 | （空） | 可选 |
| **密码** | MQTT 认证密码 | （空） | 可选，仅会话存储 |
| **心跳间隔** | Keep-alive 时间 | 60 秒 | 10-300 秒范围 |

**弹窗动画** - bounceInCenter 动画：
- 0% - 从中心缩小至 0.3 倍，透明
- 50% - 放大至 1.05 倍，产生弹性感
- 100% - 回到正常大小（1 倍）
- 用时 0.5s，cubic-bezier 缓动

**功能按钮**：
- 🧪 **测试连接** - 6 秒超时验证连接
- 💾 **保存配置** - 保存到 LocalStorage
- ✅ **应用配置** - 连接成功后自动关闭弹窗

### 4. 菜单操作

**菜单项目**：
1. **MQTT 连接配置** - 打开配置弹窗
2. **清空历史数据** - 重置所有图表数据
3. **图表显示设置** - 预留功能（可扩展）
4. **导出监控数据** - 预留功能（可扩展）
5. **关于系统** - 预留功能（可扩展）

**菜单交互**：
- 悬停时文字变蓝，向右缩进
- 光线从左到右扫过
- 有分割线区分功能组

### 5. 连接状态指示

**状态点** - 左上角绿色圆点：
- **已连接** - 绿色，2 秒周期发光
- **已断开** - 红色，1 秒周期发光
- **悬停** - 显示详细状态文本

## 🚀 快速开始

### 1. 打开页面

在浏览器中打开 `index.html` 文件即可看到应用界面。

### 2. 配置 MQTT 连接

- 点击右上角菜单按钮（☰）
- 选择 "MQTT 连接配置"
- 填入您的 MQTT 服务器信息：
  ```
  服务器地址: wss://your-mqtt-server.com:8084/mqtt
  客户端ID: web-monitor-001
  订阅主题: environment/data
  ```
- 点击 "测试连接" 验证参数
- 点击 "应用配置" 开始连接

### 3. 查看实时数据

连接成功后，应用将自动：
1. 订阅您指定的 MQTT 主题
2. 接收环境数据并实时显示
3. 绘制数据趋势图表
4. 状态点显示绿色发光表示已连接

## 📡 MQTT 数据格式

系统期望接收的 MQTT 消息格式（JSON）：

```json
{
    "temperature": 25.5,      // 温度（℃）
    "humidity": 60,           // 湿度（%）
    "windSpeed": 2.3,         // 风速（m/s）
    "illumination": 1200      // 光照强度（lux）
}
```

**说明**：
- 所有字段都是可选的（存在时才会更新对应显示）
- 数值为 0 时也会被显示（不作为无效值）
- 时间戳由系统自动添加

## ⚙️ 配置和定制

### 修改默认配置

编辑 `js/mqtt-config.js` 中的 `DEFAULT_CONFIG` 对象：

```javascript
const DEFAULT_CONFIG = {
    host: 'wss://mb67e10b.ala.cn-hangzhou.emqxsl.cn:8084/mqtt',
    clientId: 'web-monitor-' + Math.random().toString(36).substr(2, 9),
    topic: 'environment/data',
    username: '',
    password: '',
    keepalive: 60
};
```

### 修改颜色方案

```css
/* 品牌蓝 */
--primary-blue: #2563eb;

/* 成功绿 */
```

编辑 `css/style.css` 中的 animation 属性：

```css

    animation: slideInUp 0.5s ease-out backwards;
    /* 改为 0.3s 加快，0.8s 减慢 */
}
```

## 🔐 安全特性

### 内容安全策略 (CSP)

```
default-src 'self'              ← 默认拒绝所有跨域
script-src 'self' CDN           ← 仅允许本地和 CDN 脚本
style-src 'self'                ← 仅允许本地样式
connect-src 'self' wss:         ← 仅允许 WebSocket 和本地
```

## 🐛 常见问题

### Q: 无法连接到 MQTT 服务器？

**A:** 检查以下几点：
1. 服务器地址格式正确（wss://...）
2. 端口号已包含在地址中
3. 防火墙允许 WebSocket 连接
4. MQTT 服务器已启动并可访问

### Q: 图表数据为什么不显示？

**A:** 可能原因：
1. MQTT 连接未建立（查看状态点颜色）
2. 收到的数据格式不正确（查看浏览器控制台）
3. 数据中缺少预期字段
4. 清空了历史数据

### Q: 动画显示很卡顿？

**A:** 解决方法：
1. 关闭浏览器其他标签页
2. 禁用浏览器扩展程序
3. 更新到最新浏览器版本
4. 检查 CPU/GPU 使用率

### Q: 如何禁用某个动画？

**A:** 在 `css/style.css` 中注释掉对应的 animation 属性：

```css
.data-card {
    /* animation: slideInUp 0.5s ease-out backwards; */
}
```
